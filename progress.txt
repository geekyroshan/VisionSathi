## Codebase Patterns

- Use design tokens from `constants/colors.ts` — never hardcode colors
- All touchable elements must have accessibilityLabel and accessibilityRole
- Use Zustand stores for state, not React context
- API responses always include processingMs for monitoring
- Moondream prompts live in `prompts/` directory, not inline
- Use Zustand v5 syntax: `create<State>()((set) => ...)`
- expo-camera uses `CameraView` not `Camera` in SDK 54
- Use `--legacy-peer-deps` for npm install if peer conflicts occur
- Ollama HTTP API at localhost:11434 for Moondream inference (not in-process torch)
- expo-speech-recognition for on-device STT (not backend whisper)
- Dynamic API URL from Zustand settings store (not hardcoded)
- Glassmorphism via rgba() backgrounds/borders (not expo-blur for performance)
- react-native-reanimated v4 for 60fps capture button animations
- StatusBar component imports Text from `@/components/ui/Text` directly (avoids circular imports)
